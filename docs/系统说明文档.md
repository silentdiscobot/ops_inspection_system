# 灵兔洞察系统（运维巡检系统）说明文档

## 1. 文档说明

本文档依据当前仓库代码整理，适用于系统使用、部署、运维和二次开发。代码仓库名称为“运维巡检系统”，Web 页面显示名称为“灵兔洞察系统”。

系统是一个单机部署的 Web 运维工具，通过 SSH 对多台服务器执行资源巡检和网关代理检测，支持任务保存、定时执行、失败重试、实时进度展示以及 Excel/PDF 报告生成。

## 2. 功能概览

- 用户登录：图片验证码、登录失败限流、Session 会话管理。
- 用户与权限：管理员、操作员、报告查看员三种角色。
- 服务器管理：服务器分组、一个服务器加入多个分组、密码或 SSH 私钥认证。
- 巡检任务：保存、编辑、删除、立即执行和按日定时执行。
- 资源巡检：采集系统运行时间、CPU、内存和 `df -hP` 返回的全部挂载点磁盘使用率；主表展示最高值，异常详情逐条展示超过阈值的挂载点。
- 网关代理检测：在指定服务器上执行受限制的 `curl` 命令，通过响应关键字判断结果。
- 执行队列：SQLite 持久化队列、多工作线程、失败重试和服务重启恢复。
- 实时进度：通过 Socket.IO 向页面推送执行进度，同时提供轮询接口。
- 报告管理：生成、查看、下载和删除 Excel/PDF 报告。
- 安全保护：CSRF 校验、密码哈希、凭据加密、安全响应头和报告路径校验。

## 3. 技术架构

| 层次 | 实现 |
| --- | --- |
| Web 服务 | Flask 3.0.3、Flask-SocketIO 5.3.6 |
| 页面 | Jinja2 模板、Bootstrap、本地 Axios/Socket.IO 静态资源 |
| 数据存储 | SQLite，WAL 模式 |
| SSH 客户端 | Paramiko 3.4.0 |
| 凭据加密 | AES-GCM，密钥由环境变量或密钥文件提供 |
| 密码存储 | Werkzeug scrypt 哈希，兼容旧 SHA-256 并在登录后自动升级 |
| 报告 | openpyxl 生成 Excel，ReportLab 生成 PDF |
| 后台任务 | 进程内调度线程与持久化队列工作线程 |

核心执行链路如下：

```text
用户手动执行 / 定时调度
          ↓
写入 inspection_runs（queued）
          ↓
队列工作线程领取任务（running）
          ↓
按分组读取服务器并解密 SSH 凭据
          ↓
资源巡检 + 可选代理检测
          ↓
保存 inspection_results 并推送实时进度
          ↓
生成 Excel/PDF 报告
          ↓
success；失败则延迟重试，达到上限后 failed
```

系统首次启动时自动创建默认管理员：

生产部署前必须修改默认密码。数据库中已经存在 `admin` 时，修改 `OPS_ADMIN_PASSWORD` 不会重置其密码，应登录系统后在个人资料页修改。

## 5. 页面操作说明

### 5.1 登录与个人资料

1. 打开系统登录页，输入用户名、密码和图片验证码。
2. 同一来源地址与用户名在 15 分钟内失败 5 次后会临时锁定。
3. 登录后可进入个人资料页修改密码或退出登录。
4. 新密码至少 8 位，并同时包含大写字母、小写字母和数字。

### 5.2 用户管理

仅超级管理员可进入“系统管理项 → 用户管理”。可创建、编辑和删除用户；系统默认管理员不可删除。

用户名仅允许字母、数字、下划线、点和短横线，长度不超过 64 个字符。

### 5.3 服务器管理

进入“服务器管理”后：

1. 创建服务器分组，例如“生产环境”“测试环境”。
2. 新增服务器，填写主机名/IP、SSH 端口、用户名和备注。
3. 选择认证方式：
   - 密码认证：填写 SSH 密码。
   - 私钥认证：粘贴 RSA、ECDSA 或 Ed25519 格式私钥；加密私钥还需填写口令。
4. 可将同一服务器加入多个分组。

服务器密码、私钥和私钥口令使用 AES-GCM 加密后写入 SQLite，服务器列表接口不会返回这些字段。

### 5.4 新建巡检任务

进入“巡检项管理”，填写：

- 任务名称：任务标识，并作为报告文件名主体。
- 项目名称：显示在报告基本信息中。
- 巡检人：显示在报告基本信息中。
- 报告格式：Excel 或 PDF。
- 服务器分组：留空时巡检全部服务器。
- 巡检项：CPU、内存、磁盘，可分别启用或停用。
- 网关代理规则：可添加多条，每条指定服务器分组、`curl` 命令和成功关键字。
- 定时执行：启用后填写每日执行时间，格式为 `HH:MM`。

点击“开始巡检”会直接创建一次手动执行；点击“保存任务”会将配置保存到任务列表。

### 5.5 执行与查看进度

进入“服务器巡检”，可以查看已保存任务和最近 50 条执行记录。任务执行后先进入队列，常见状态为：

| 状态 | 说明 |
| --- | --- |
| `queued` | 等待工作线程执行，或失败后等待重试 |
| `running` | 正在执行巡检 |
| `success` | 巡检和报告生成成功 |
| `failed` | 达到最大尝试次数后仍失败 |

页面通过 Socket.IO 接收实时进度；进度也会写入数据库和 `logs/progress.json`。

### 5.6 历史报告

“历史报告单”展示 `reports/` 目录内的 `.xlsx` 和 `.pdf` 文件，并按文件修改时间倒序排列。所有登录用户可预览和下载，管理员及操作员可删除。

## 6. 巡检实现与判定规则

### 6.1 资源采集命令

远程 Linux 服务器通过 SSH 执行以下命令：

| 指标 | 命令或策略 |
| --- | --- |
| 运行时间 | 优先读取 `/proc/uptime`，仅显示最大的时间单位 |
| CPU | 对 `/proc/stat` 进行两次采样计算，失败时回退到 `top` |
| 内存 | 读取 `/proc/meminfo` 的 `MemTotal` 和 `MemAvailable`，兼容无 `MemAvailable` 的旧内核 |
| 磁盘 | `df -hP`，不做过滤地读取全部挂载点；主表展示最高使用率，异常详情列出超过阈值的挂载点 |

当目标是回环地址时，系统直接检查本机，不要求本机开启 SSH。macOS 本机内存使用率通过 `sysctl` 和 `vm_stat` 计算。

### 6.2 状态判定

默认告警阈值如下：

- CPU 使用率大于 80%。
- 内存使用率大于 80%。
- 磁盘使用率大于 80%。

任一启用指标超过对应阈值，服务器状态为“告警”；SSH 或采集失败时为“连接失败”；其余为“正常”。比较逻辑使用“大于”，等于阈值不会触发告警。

### 6.3 网关代理检测

系统通过 SSH 在目标服务器执行 `curl`，输出中包含成功关键字即判定成功。为降低命令注入和文件读写风险：

- 只允许 `curl` 和 HTTP/HTTPS 地址。
- 禁止分号、换行、反引号、命令替换、管道及逻辑运算符。
- 禁止输出文件、上传文件、配置文件、跟踪文件等危险选项。
- 禁止 `file://`、FTP、SCP、SFTP 以及 `@本地文件` 参数。

## 7. 安装与启动

### 7.1 运行环境

- Python 3.11 或兼容版本。
- 对被巡检服务器具有网络访问权限。
- 被巡检 Linux 服务器已开启 SSH，并提供可用凭据。
- 生成中文 PDF 时，系统需要可用的中文字体；Dockerfile 已安装文泉驿字体。

### 7.2 本机启动

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

export OPS_SECRET_KEY='请替换为足够长的随机值'
export OPS_ADMIN_PASSWORD='请替换为强密码'
export OPS_AES_KEY='base64编码的32字节随机密钥'

python app.py
```

生成 AES 密钥示例：

```bash
python3 -c "import os,base64; print(base64.b64encode(os.urandom(32)).decode())"
```

默认监听 `0.0.0.0:1999`，浏览器访问 `http://服务器地址:1999`。

### 7.3 Docker 启动

```bash
docker build -t ops-inspection-system .

docker run -d \
  --name ops-inspection-system \
  -p 1999:1999 \
  -e OPS_SECRET_KEY='请替换' \
  -e OPS_ADMIN_PASSWORD='请替换' \
  -e OPS_AES_KEY='请替换' \
  -v ops-data:/app/data \
  -v ops-reports:/app/reports \
  -v ops-logs:/app/logs \
  ops-inspection-system
```

生产模式下建议设置 `OPS_DEBUG=0`。若 Session 密钥、管理员初始密码或 AES 密钥仍使用默认配置，应用会记录安全警告但不会拒绝启动。

## 8. 环境变量

登录 Cookie 固定支持普通 HTTP，不需要额外环境变量。登录会话空闲 2 小时后失效；有持续操作时自动续期且累计时长不设上限。Cookie 仅在当前浏览器会话内有效，关闭浏览器后重新打开必须登录。

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `OPS_HOST` | `0.0.0.0` | 服务监听地址 |
| `OPS_PORT` | `1999` | 服务端口 |
| `OPS_DEBUG` | `1` | 是否启用开发模式，`1` 为启用 |
| `OPS_SECRET_KEY` | `change-me` | Flask Session 签名密钥 |
| `OPS_ADMIN_PASSWORD` | `Admin@123` | 首次创建默认管理员时使用 |
| `OPS_AES_KEY` | 内置开发密钥 | AES 密钥，支持 Base64 或十六进制 |
| `OPS_AES_KEY_FILE` | 空 | AES 密钥文件路径，优先于 `OPS_AES_KEY` |
| `OPS_SSH_STRICT_HOST_KEY` | 开发模式为 `0`，生产模式为 `1` | 是否拒绝未知 SSH 主机密钥 |
| `OPS_QUEUE_WORKERS` | `2` | 队列工作线程数，最小为 1 |
| `OPS_QUEUE_RETRY_DELAY` | `10` | 任务失败后的重试等待秒数 |
| `OPS_CPU_THRESHOLD` | `80` | CPU 告警阈值（百分比） |
| `OPS_MEM_THRESHOLD` | `80` | 内存告警阈值（百分比） |
| `OPS_DISK_THRESHOLD` | `80` | 磁盘告警阈值（百分比） |

注意：AES 密钥必须在整个数据生命周期内保持不变，否则数据库中已有服务器凭据将无法解密。建议通过受控密钥文件或密钥管理系统注入，不要写入代码或镜像。

## 10. 队列与定时任务机制

- 每次执行生成唯一 `run_id` 并写入 SQLite。
- 默认启动 2 个工作线程，可并发处理不同任务；单个任务内部逐台服务器执行。
- 每次任务最多尝试 2 次，失败后默认等待 10 秒再试。
- 服务启动时，未超过最大尝试次数的 `running` 任务会恢复为 `queued`；已达到上限的任务会标记为 `failed`。
- 调度器每 5 秒检查一次任务，按应用服务器本地时区匹配 `HH:MM`。
- 定时任务使用“任务 ID + 当前分钟”去重，同一分钟不会重复入队。

当前调度器和工作线程都运行在 Web 进程内，因此生产环境只应启动一个应用进程。若同时启动多个独立应用进程，虽然数据库去重能避免部分重复任务，但每个进程都会启动自己的调度线程，部署模型不符合当前设计。

## 11. 安全说明

- 所有修改类请求除登录外均要求 `X-CSRFToken` 请求头。
- Session Cookie 开启 `HttpOnly` 和 `SameSite=Lax`；生产环境默认要求 `Secure`。
- 响应设置 CSP、禁止 iframe、禁止 MIME 嗅探等安全头。
- 密码采用带盐 scrypt 哈希；旧 SHA-256 密码在成功登录后自动升级。
- SSH 凭据使用 AES-GCM 加密，API 返回服务器信息时移除凭据字段。
- 报告下载、预览和删除均限制在 `reports/` 目录，且仅允许 `.pdf`、`.xlsx`。
- 生产环境建议在系统前部署 HTTPS 反向代理，并限制来源网络和 1999 端口访问范围。
- 开启 `OPS_SSH_STRICT_HOST_KEY=1` 后，应提前将目标服务器主机密钥加入运行账户的 `known_hosts`。

## 14. 常见问题排查

### 14.1 生产模式无法启动

检查日志中是否提示不安全配置。生产环境建议设置非默认的 `OPS_SECRET_KEY`、`OPS_ADMIN_PASSWORD` 和 `OPS_AES_KEY`/`OPS_AES_KEY_FILE`，未设置时应用仍可启动。

### 14.2 服务器连接失败

依次检查网络连通性、SSH 端口、用户名、认证方式和防火墙。严格主机密钥校验开启时，还需确认 `known_hosts` 中存在正确的服务器公钥。

### 14.3 CPU 始终显示 0

确认目标机存在 `top`；系统还会尝试 `mpstat` 和 `sar`。若目标系统输出格式与常见 Linux 格式不同，需要扩展 `inspection.py` 中的解析逻辑。

### 14.4 PDF 中文显示异常

确认操作系统已安装文泉驿、Noto CJK、宋体、黑体等可用中文字体，并通过 `fc-list` 检查。Docker 镜像构建时已安装常用中文字体。

### 14.5 修改 AES 密钥后服务器全部巡检失败

原凭据只能由原 AES 密钥解密。恢复原密钥，或删除服务器记录并使用新密钥重新录入凭据。

### 14.6 定时任务没有执行

确认任务已启用、时间格式为 `HH:MM`、应用服务器时区正确，并检查 `logs/app.log` 中调度器和队列线程日志。定时任务按应用所在主机的本地时间运行。
